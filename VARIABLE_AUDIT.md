# Variable Collision Audit — opencode-bybit

Generated by: Phase 5 (evidence-driven)
Date: 2026-06-20

PowerShell is case-insensitive. The following case-variant variable names were found across the codebase.

---

## Finding 1: `$tp` / `$tP` / `$TP` / `$Tp`

PowerShell treats all these as the same variable. This is a known bug (documented in AGENTS.md as Bug #1).

| File | Line | Usage |
|---|---|---|
| `paper_trading/run_paper_trader.ps1` | 48 | `$symCfg.TP` (property access, but convention inconsistent) |
| `paper_trading/run_paper_trader.ps1` | 231-237 | `$tp` used as loop variable in tpResults |
| `icp_fullgrid.ps1` | 208 | `$tps` array definition |
| `icp_fullgrid.ps1` | 230-237 | `$tp` as loop variable + `$tpT` for take-profit price |
| `icp_complete.ps1` | 333 | `$tps` array + `$tp` inside $tpResults loop |
| `icp_deep_dive.ps1` | 292 | Same pattern |
| `xrp_complete.ps1` | 346 | Same pattern |
| `sol_bybit_backtest.ps1` | 69 | `$tp=0.5` (parameter) |
| `sol_3month_sim.ps1` | 63 | `$tp=0.5` |

**Evidence of collision**: In `icp_fullgrid.ps1:230`, `$tp` is a loop variable iterating TP values while simultaneously `$tpResults` accumulates results. The `$tpResults` variable name includes `$tp` prefix but since `$tp` is also reused as a loop iterator, this creates a naming collision within the same scope.

**Risk**: LOW (separate scopes in functions), but naming convention is inconsistent.

## Finding 2: `$c` (close prices) / `$c` (candidates)

## Finding 5: `$i` (candle index) reused across nested loops

Multiple files reuse `$i` as a loop index across nested scopes (e.g., `icp_fullgrid.ps1:71` outer loop, `:154` inner loop). PowerShell's scoping means each `for` creates its own scope, so this is technically safe, but it's a readability issue.

**Risk**: LOW
