## Finding 1: RSI Bruteforce on Full Dataset (CRITICAL)

**Evidence**: `icp_fullgrid.ps1:91-99`

```powershell
foreach($per in (5..50|?{$_%3-eq2-or$_-eq5})){
    $r=Calc-RSI $c $per;$pb=$null;$pbs=0
    foreach($ob in $obs){foreach($os in $oss){
        ...
        for($i=$per;$i-lt$c.Count-3;$i++){
            # Tests signal on ALL AVAILABLE DATA
        }
        ...
        $score=$wr*$t;if($score-gt$pbs){$pbs=$score;$pb=...}
    }}
}
```

**Root cause**: The `$c` array contains ALL available klines. There is NO train/test split. The same data used to find the best RSI parameters is also used to report the win rate of those parameters.

**Impact**: Parameters are fit to noise patterns in the full dataset. Reported win rates are in-sample and will NOT reproduce out-of-sample.

**Same pattern in**: `icp_complete.ps1:217-248`, `icp_deep_dive.ps1:148-168`, `xrp_complete.ps1:217-248`

## Finding 5: Skip-After-Loss Artificially Inflates WR (HIGH)

**Evidence**: Multiple files (now removed per AGENTS.md):

The skip-after-loss logic selectively avoids taking the next trade after a loss. This is not true data leakage in the time-series sense, but it creates a survivorship-like bias where losing streaks are undercounted.
